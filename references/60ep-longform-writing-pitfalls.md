# 60集长线剧本创作经验与陷阱（2026-06-27会话产出）

> 基于《代码里的长安》8集→60集扩展实战，含10轮自检19项缺陷修复

## 一、创作流程铁律

### 1.1 正文创作禁止使用子程序

```
❌ delegate_task(goal='写20集剧本') 
   → 子程序不掌握全局人物弧线，写出的人物在中段弧线断裂
   → 子程序不理解情感节奏分布，情感点扎堆或空白
   → 尾钩同质化严重，修复成本远高于直接写

✅ 主agent逐集直接撰写
   → 每集3-5场戏，全局把控人物弧线
   → 情感点按波浪曲线分布
   → 尾钩差异化设计，每集不同悬念类型
```

适用子程序的任务：查资料、校验格式、分段校对
不可委托的任务：正文创作、人物弧线规划、情感节奏设计

### 1.2 角色名统一工作流

当需要批量修改角色名时（如林小禾→林依依、韩瑾→延炎），需同步修改以下文件：
1. gen_60_script.py（剧本生成脚本，数据结构中的字符串）
2. 生成的剧本草稿.md（直接文本替换）
3. 参赛剧本.docx（python-docx逐段逐run替换）
4. 故事大纲.docx（同上）
5. 完整参赛稿.md（直接替换）
6. 10轮自检报告.md（直接替换）

注：Word文档必须逐run替换而非仅paragraph.text，否则粗体会丢失。

## 二、60集结构关键参数

### 2.1 五段结构（基于短剧大师模板适配）

| 段落 | 集数 | 集数占比 | 核心任务 |
|:----|:----:|:--------:|:---------|
| 建立期 | E01-E10 | 16% | 日常→异常→金手指→首个反派线索 |
| 展开期 | E11-E22 | 20% | 密室/真相/组织渗透/配方线索 |
| 升级期 | E23-E38 | 27% | 记忆深渊/全面入侵/代价显现 |
| 转折期 | E39-E52 | 23% | 6%缺口/角色站队/反派转变/三线汇合 |
| 终局期 | E53-E60 | 14% | 对峙/救赎/展览/和解 |

### 2.2 情感节奏分布

情感点总量控制：60集约12-15个情感高点（泪点/燃点）

| 段落 | 情感点数量 | 间距控制 |
|:----|:---------:|:---------|
| E01-E12 | 3-4个 | 每3-4集一个，建立角色绑定 |
| E13-E30 | 4-5个 | 信息揭露+情感释放交替 |
| E31-E45 | 3-4个 | 连续压力区，情感点密集 |
| E46-E60 | 3-4个 | 逐级上升至终极高潮 |

常见陷阱：
- **前松后紧**：前20集情感点偏少(2-3个)，中段11集挤6个 → 观众在前期流失
- **暗线盲区**：外婆线在E05-E16中断11集 → 每5-8集必须出现一次暗线信号（哪怕1句台词）

### 2.3 尾钩差异化五类轮换

| 尾钩类型 | 效果 | 示例 |
|:---------|:-----|:-----|
| 信息揭露 | 「原来/其实/竟是」 | 延炎不是为逃跑死的 |
| 情感爆点 | 泪点/燃点 | 这句话她记了一千二百年 |
| 危机降临 | 悬念驱动追更 | 有人在碰壁画，有蓝光 |
| 身份反转 | 认知颠覆 | 外婆不是离职是被灭口的 |
| 关系突变 | 人物关系变化 | 何砚秋在保护她 |

每个尾钩必须同时满足：驱动追更 + 不重复前集同类。

### 2.4 批量修复策略

对60集剧本做批量修改时，不要逐个patch剧本生成脚本：
1. 先生成草稿.md
2. 直接用replace()做批量文本替换（省时且不易出错）
3. 从草稿.md重新生成Word文档

修复优先级：P0(尾钩/暗线/节奏) → P1(情感点/视觉/弧线) → P2(感官细节/对白优化)

## 三、本会话修复清单参考

19项缺陷的实际修复效果：
- 情感点：修复前60集共14个 → 修复后18个，分布更均匀
- 强尾钩比例：80% → 95%
- 暗线覆盖率：从6个节点 → 12个节点，覆盖全部五段
- 总中文字数：7,020 → 7,941 (+13%)
