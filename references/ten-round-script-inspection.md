## 四、剧本缺陷修改工作流

从《代码里的长安》13次patch实战中提炼：

### 4.1 修改顺序

```yaml
优先顺序:
  P0: 🔴致命错误 — 时间线/技术设定/逻辑悖论（必须先修，影响评审一票否决）
  P1: 🟠节奏问题 — 缺反转/太慢/反派转变生硬（中段集数的核心问题）
  P2: 🟡情感增强 — 加闪回/回扣/名场面强化（好上加好）
  P3: 🟢排版清理 — 删重复/补暗示/消除歧义
```

### 4.2 patch技巧（适用于长剧本）

```
1. 先确认精确位置（用 read_file offset 定位行号）
2. 用 patch 做 targeted replace，带3行以上上下文确保唯一性
3. 大段新增先用附近的一段原文作为 old_string 锚点
4. 每改完一个 grep 验证零残留：
   grep -c "旧文本" file.md    # 应 = 0（已删除）或 = 正确次数（已更新）
5. 所有 patch 完成后一次性验证：
   - 时间线一致性
   - 角色名字拼写
   - 提交清单完整性
```

### 4.3 剧本修复清单模板

```markdown
| 编号 | 严重级 | 原始问题 | 根因 | 修改位置 | 完整修复动作 | 验收标准 |
|:----:|:-----:|:---------|:-----|:--------|:-----------|:--------|
```

## 五、常见致命陷阱（从实战中提炼）

1. **时间悖论陷阱** — 角色说"等了一千二百一十八年"，文末字幕写"公元755-2026，一千二百七十一年" → 必须统一
2. **承诺集体陷阱** — 大纲写"十二集体量"但实际只写了8集 → 必须精确匹配
3. **反派转变生硬陷阱** — 只说了一通话就弃恶从善 → 需要外部触发（国际刑警/账户冻结/家庭线索）
4. **回扣缺失陷阱** — 角色在第5集提出哲学问题，到结局没有答案 → 第8集必须回扣
5. **死亡抽象陷阱** — 重要角色之死只有口头交代 → 必须有具象画面（史书残卷/闪回/文物证据）
