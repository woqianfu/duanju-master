# 多集版本统一方法论

> **发现日期**：2026-06-15，实战《苍生》E41-E60版本断裂修复  
> **严重度**：🔴 致命（角色团突变=叙事不可修复）  
> **适用范围**：多集剧本审查中检测并修复版本断裂

## 问题描述

当剧本采用多版本迭代创作时，可能在某个集段分界点发生「版本断裂」——前一版本的角色、体系、代价路径与后一版本完全不同，且无任何过渡。

### 典型症状

| 断裂点 | 前一版本 | 后一版本 | 变化 |
|--------|---------|---------|------|
| 角色团 | A/B/C/D/E/F 六人 | W/X/Y/Z 四人 | 完全不同的角色名 |
| 核心体系 | 锁妖环/六芒星 | 苍生塔/七环 | 世界观术语突变 |
| 代价路径 | 逐步感官剥夺 | 一次性献祭 | 叙事逻辑断裂 |
| 反派身份 | 算法型外星反派 | 三百年宿敌复仇者 | 动机不可调和 |

### 根本原因

1. **硬拼接**：不同创作阶段的版本被直接拼接，未做过渡处理
2. **重大分界点效应**：通常发生在「40集」「50集」等大的段落分界处
3. **前后创作人/模型不同**：不同session产出的内容世界观不一致

## 检测方法

### SQ-VERSION 扫描流程

```bash
# 1. 提取每集的「在场人物」列表
grep "在场人物\|【出场】" E*.md > characters.txt

# 2. 比对相邻集角色团
# 任意两个角色的名字完全不同 + 人数变化 → 版本断裂

# 3. 提取每集的「体系关键词」
grep -c "锁妖环\|六芒星\|苍生塔\|七环" E*.md

# 4. 若前20集用词A、后20集用词B → 体系断裂
```

### 自动检测脚本

```python
import re, glob

files = sorted(glob.glob("E*-*.md"))
prev_chars = set()

for f in files:
    text = open(f).read()
    # 提取角色名（从【出场】或 在场人物 行）
    m = re.search(r'【出场】(.*?)(?:\n|$)', text)
    if not m:
        m = re.search(r'在场人物[：:](.*?)(?:\n|$)', text)
    if m:
        chars = set(re.findall(r'[^\s、，,·]+', m.group(1)))
        if prev_chars and not (chars & prev_chars):
            print(f"⚠️ 版本断裂: {f} — 角色团从 {prev_chars} 变为 {chars}")
        prev_chars = chars
```

## 修复决策树

```
发现版本断裂
  │
  ├─ 两版都是完整剧情？
  │   ├─ 是 → 选一版作为正本，重写另一版
  │   └─ 否 → 以完整版为正本
  │
  ├─ 前版角色更丰富（6人 vs 5人）？
  │   ├─ 是 → 以前版为本（选项A）— 重写后段
  │   └─ 否 → 以后版为本（选项B）— 重写前段
  │
  └─ 后版有更完整的终局？
      ├─ 是 → 提取后版情感骨架，套入前版体系重写
      └─ 否 → 纯原创补齐缺失段
```

### 选择策略

| 维度 | 权重 | 检查方法 |
|------|:----:|---------|
| 人物连续性 | 🔴最高 | 哪一版的角色团与前后连贯？ |
| 代价路径一致性 | 🔴最高 | 哪一版的代价机制可追溯到E01？ |
| prompt格式完整度 | 🟡中 | 哪一版每镜有完整标注？ |
| 终局完整度 | 🟡中 | 哪一版有完整结局？ |

## 实战案例：《苍生》E41-E60修复

### 决策

选择 **选项A**：以 E41-E43 为本（六人团+锁妖环+逐步退化），重写 E44-E60（17集）

### 理由
1. **人物连续性**：六人团（延炎/林依依/张猎户/孙九/赵铁匠/小鹿）是E01-E40一砖一瓦搭的
2. **代价连续性**：逐步退化路径（尾指颤→虎口裂→...→左腿废）有完整退化链条
3. **后版终局可迁移**：E44-E60的情感骨架（塔沉→代价归零→六人碑→铁锈不锈）可翻译到锁妖环体系

### 执行
- 三路delegate_task并行重写（E44-E49 / E50-E55 / E56-E60）
- 每路6/6/5集，总17集，720KB
- 输出完整prompt标注格式
- 最终全量验证：角色/体系/代价/暗线四项一致

## 与SQ系统关联

- **SQ-VERSION**：Phase 4新增审查关卡，在SQ0前运行
- **SQ5**：跨集一致性检查——角色团突变=暗线断裂=不通过
- **SQ4**：逻辑连贯性——版本断裂=因果断裂=五类断裂全部命中

## 预防措施

1. **统一术语表**：开写前列出所有核心体系术语，全程锁定
2. **角色团锁定**：前10集确定阵容后，全季不可变更
3. **分集大纲锁定后不改**：大纲定稿后，后续集数的体系锚定不可变
4. **跨session创作需传角色表**：每次新session，先读已写集数的角色体系状态
