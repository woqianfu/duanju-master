| 音视频同步 | 单次生成同步输出唇形同步对话、匹配音效、环境音景和 BGM |
| 视频编辑 | 替换角色、添加/去除物体、风格迁移（自然语言驱动） |

| 角色一致性 | 跨帧/跨镜头/跨角度保持面部、服装、产品 Logo 一致 |
| 一镜到底 | 跨越多个场景生成无剪辑的长画面 |

### R.2.2 技术参数

| 参数 | 推荐值 | 说明 |
|------|:--:|------|
| 视频时长 | 5-10 秒（新手） / 4-15 秒（可选） | 新手别超过 8 秒，容易崩 |
| 画幅比例 | 16:9（电影感）/ 9:16（短视频）/ 1:1 | 按平台选择 |
| 创意度 | 0.3-0.6 | 太高容易跑偏 |
| 运动强度 | 中等 | 高强度适合快节奏打斗，低强度适合慢镜 |
| 参考强度 | 0.6-0.85 | 太高死板，太低忽略参考 |
| 种子 | 固定种子可复现，随机留空 | 用于 A/B 测试 |

### R.2.3 重要限制

| 限制 | 说明 |
|------|------|
| ❌ 不支持负面提示词 | 只写想看到的，不写不想看到的 |
| ❌ 无 negative prompt 字段 | "不要模糊"→"清晰锐利"；"不要美颜"→"真实皮肤纹理" |
| @参考上限 | 最多 9 张图 + 3 段视频（总≤15秒）+ 3 段音频（总≤15秒） |
| 总时长限制 | 多段视频拼接总时长 ≤ 15 秒 |
| prompt 建议长度 | ≤ 8000 字符（小云雀建议上限） |

## R.3 Seedance 2.0 提示词体系

### R.3.1 五要素导演法（权重从高到低）

```
1. 主体/角色（谁/什么） → 最重要，先写清楚人/物特征、服装、年龄、表情
2. 动作/剧情（做什么） → 核心叙事，用现在时 + 具体动词
3. 镜头/运镜（怎么拍） → 专业电影术语或白话描述运动、角度、时长
4. 场景/氛围/光影（在哪里/什么感觉） → 环境 + 天气 + 色调 + 情绪
5. 风格/特效/音频（像什么电影/加什么效果） → 电影名、导演风格、音效提示
```

### R.3.2 万能起手公式

```
[风格类型] + [主体1][状态1] + [主体2][状态2] + [环境描写] + [光影气氛] + [镜头运镜]
```

### R.3.3 最稳公式模板（可直接复制改）

```
[主体详细描述]，[动作/剧情详细拆分]，[镜头语言 + 时间轴拆分]，[场景环境 + 光影氛围]，[电影风格/特效/音频提示]，高质量，电影感，物理真实，角色一致性强
```

### R.3.4 时间轴拆分法

```
[00-05s] 内饰特写：老司机专注眼神，雨水打在挡风玻璃
[05-10s] 切到年轻挑战者紧握方向盘，呼吸急促
[10-15s] 绿灯亮起，两车同时起步，水花四溅，高速跟拍
```

### R.3.5 @参考标签用法

| 标签 | 用法 | 说明 |
|------|------|------|
| `@Image1` | 引用上传的第 1 张图片 | 编号从 1 开始，按上传顺序 |
| `@Video1` | 引用上传的第 1 段视频 | 提取动作/运镜/特效 |
| `@Audio1` | 引用上传的第 1 段音频 | 控制节奏/情绪起伏 |

**组合示例**：
```
参考 @视频1 的人物动作和镜头语言，生成 @图片1 和 @图片2 的打斗场面，打斗背景是 @图片3，背景音乐是 @音频1
```

### R.3.6 运镜指令速查

| 运镜 | 指令写法 | 适合场景 |
|------|------|------|
| 固定镜头 | `固定机位，中景，背景虚化` | 对话/日常 |
| 缓慢推进 | `从中景到特写的慢速推进，持续5秒` | 情绪转折/揭示 |
| 跟拍 | `侧面跟拍，与主体同速移动` | 行走/追逐 |
| 摇移 | `从 A 摇移到 B，缓慢平滑` | 环境展示 |
| 俯拍 | `俯拍吊臂镜头，下降到视线高度` | 登场/威慑 |
| 环绕 | `环绕主体 360° 弧线运动` | 三维展示 |
| 手持感 | `手持纪实风格，微妙自然晃动` | 真实感 |
| 希区柯克 | `希区柯克变焦，主体大小不变背景压缩` | 心理悬念 |

## R.4 角色一致性完全指南

### R.4.1 一致性四大类型

| 类型 | 含义 |
|------|------|
| 脸部一致性 | 同一张脸始终是同一张脸 |
| 产品一致性 | 同一产品保持同样几何形态 |
| 手部一致性 | 同一个手部交互看起来可信 |
| 场景一致性 | 同一个场景维持同一套视觉逻辑 |

### R.4.2 一致性工作流

```
1. text-to-video 做概念探索
2. image-to-video 锁首帧
3. reference-to-video 锁身份或产品
```

### R.4.3 角色一致性黄金三件套

```
1. 三视图参考图（正面/侧面/背面）→ 上传为 @Image1,2,3
2. 固定外貌描述（写入每镜 prompt）→ "参考 @Image1 的面部特征，保持发型/服装一致"
3. 保持镜头简单 — 说话时不加复杂肢体动作和转场
```

### R.4.4 不变量清单

在写 prompt 前，先把"不能变的东西"列出来：
- 脸部身份
- 服装
- 色彩基调
- 背景布局
- 手和物体的关系

### R.4.5 一致性排查顺序

1. 有没有先定义必须固定的部分？
2. 当前模式是不是太开放？
3. 运镜是不是太激进？
4. 主体在画面里是不是太小？
5. 多个 prompt 或 reference 是不是互相打架？

### R.4.6 高频问题与修复

| 问题 | 常见原因 | 修复 |
|------|------|------|
| 脸漂 | 缺少强锚点，或动作太大 | 切到 reference 模式，缩小表演幅度 |
| 产品形变 | orbit/splash/反光太复杂 | 缩短运动，优先保护几何 |
| 手一碰就坏 | 手势太复杂或主体太小 | 简化手部姿态，放大主体 |
| 背景乱跳 | 场景过载 | 删掉次要道具，降低环境复杂度 |

### R.4.7 不要试图靠这些"买来一致性"

- ❌ 更多形容词
- ❌ 彼此冲突的更多 reference
- ❌ 更长、包含多场景的 prompt
- ❌ 更大的镜头运动

> **一致性的提升来自于减少歧义，而不是增加信息体积。**

## R.5 分镜与画面控制

### R.5.1 AI 分镜 7 大核心要素

| # | 要素 | 说明 |
|:--:|------|------|
| 1 | 景别 | 远景（环境）→ 中景（互动）→ 特写（细节/情绪） |
| 2 | 角度 | 平视（客观）→ 仰视（压迫/英雄）→ 俯视（渺小/孤独） |
| 3 | 构图 | 三分法（平衡）/ 对角线（动态张力）/ 中心对称（庄重） |
| 4 | 光影 | 侧光（立体感）/ 逆光（剪影）/ 柔光（浪漫）/ 硬光（紧张） |
| 5 | 色调 | 暖色调（温馨/回忆）/ 冷色调（压抑/科幻）/ 低饱和（纪实） |
| 6 | 动势 | 明确物体运动轨迹与方向，静态分镜也具"流动感" |
| 7 | 转场 | 硬切 / 叠化 / 匹配剪辑 / 淡入淡出 / 划像 |

### R.5.2 分镜制作全流程

```
Step 1 剧本分析   → 提取关键情节 + 理解人物情绪 + 标记视觉符号
Step 2 场景拆分   → 明确每个空间的视觉属性 + 确定画面数量
Step 3 镜头设计   → 镜号/景别/角度/画面描述/时长/台词+音效/备注
Step 4 提示词撰写 → 用五要素公式逐镜翻译
Step 5 AI 生成     → 固定形象（一致性控制）+ 画面生成（精准出图）
```

### R.5.3 镜头四级分类（BX Protocol 适配 Seedance）

| 级别 | 定义 | 每集标准 | 失败处理 |
|:--:|------|:--:|------|
| S·叙事镜 | 交代"谁做了什么、结果是什么" | ≥5 个 | 失败→必须重做 |
| A·支撑镜 | 补充空间关系和情绪 | ≥4 个 | 失败→可降级为辅助镜 |
| B·辅助镜 | 衔接过渡、手部/脚步/门把手细节 | ≥4 个 | 失败→可替换 |
| C·风格镜 | 氛围粒子、雾/光/雨/尘埃 | ≤4 个 | 失败→可删除 |

| 你想要的效果 | 用这个模式 | 原因 |

## R.7 Seedance 2.0 常见避坑

### R.7.1 5 个核心技巧

1. **要具体** — 模糊描述 = 浪费积分。"一位女性走在城市中"→"一位 30 岁日本女性，白色亚麻西装，在涩谷窄巷中自信行走"
2. **先 5 秒测试** — 在花积分生成 10-15 秒前，先用 5 秒测试提示词，快速迭代 2-3 次
3. **图生视频：描述运动而非图片内容** — 图片已告诉模型"画什么"，文字只告诉它"怎么动"
4. **用 @Reference 系统** — 从 2-3 个参考文件开始，别一次用满 12 个
5. **每次只改一个变量** — 变脸→回参考图；运镜乱→改固定镜头；不全面重写

### R.7.2 5 个常见失败模式

| 失败 | 原因 | 修复 |
|------|------|------|
| 角色变脸/服装错误 | 缺少强锚点图 | 重新上传角色三视图，提高参考强度 |
| 运镜乱飞 | 运镜叠加或太激进 | 降级为"固定镜头"或"缓慢推进" |
| 动作失败/肢体扭曲 | 复合动作超出模型能力 | 拆成单一动作镜头 |
| 场景跳变 | 未锁定空间结构 | 加注"保持空间结构一致" |
| 光影闪烁 | 光源描述矛盾 | 指定单一光源方向 |

### R.7.3 单变量修复法

```
问题 → 只改一个变量（不管其他的）：
- 变脸    → 只改参考图
- 运镜乱  → 只改运镜指令
- 动作崩  → 只拆动作
- 场景跳  → 只锁场景
- 光影闪  → 只定光源
```

## R.9 提示词污染检测清单（Seedance 适配）

> 组装 prompt 后逐项扫描。任一命中→修复后重新组装。

| 污染类型 | 检测方法 | 修复 |
|------|------|------|
| 运镜叠加 | 同一 prompt 含 ≥2 种运镜（推+摇+跟…任意两种） | 只保留一种 |
| 动作叠加 | 同一 prompt 含 ≥2 个核心动作（转身+跑+拔剑…任意两种） | 拆成多镜 |
| 光线矛盾 | "白天"+"夜景" / "暖光"+"冷光" / "逆光"+"顺光" | 只保留一项 |
| 场景跳变 | "室内"+"室外" / "白天"+"黑夜" | 只保留一个场景 |
| 角色过载 | 同一 prompt 含 ≥3 个命名角色 | 拆成 ≤2 角色/镜 |
| 约束超量 | 约束项 ≥8 条 | 精简至 ≤5 条 |
| 负面表述 | "不要/禁止/避免/别" | 转为正向表述 |
