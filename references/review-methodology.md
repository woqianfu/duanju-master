# 全剧批量审查方法论

> **验证日期**：2026-06-15，60集《苍生》实战  
> **跨体裁验证**：2026-06-15，60集《星河里唐宫夜宴》（古穿今美食爱情）实战  
> **核心发现**：三路并行delegate_task + 三阶段（审查→修复→验证）是最优模式

## 审查工作流

```
阶段1：审查（3路并行，每路20集）
  子代理A：E01-E20 → 逻辑+兼容性+格式
  子代理B：E21-E40 → 逻辑+兼容性+格式
  子代理C：E41-E60 → 逻辑+兼容性+格式
  汇总 → 统一报告

阶段2：修复（按优先级）
  P0：阻断性问题（版本断裂/封印回退）
  P1：严重问题（代价跳跃/破折号句式）
  P2：中等问题（信息过晚揭示）
  P3：低优先级（格式统一）

阶段3：验证（3路并行）
  子代理A：验证E01-E25
  子代理B：验证E26-E43
  子代理C：验证E44-E60 + 全程暗线
```

## 审查清单（SQ0-SQ12速查）

### SQ0 预检查
- [ ] 文件存在且格式正确
- [ ] 识别正式版vs旧版（AIGC元数据）
- [ ] 重复版本去重归档
- [ ] 全集集数连续

### SQ1-SQ4 逻辑审查
- [ ] 封印完整度全程递减不回升
- [ ] 代价百分比全程递减不回升
- [ ] 角色团前10集锁定全程不变
- [ ] 场景空间锚点物位置尺寸一致
- [ ] 尾钩在下一集得到回应
- [ ] 时间线标注无跳跃

### SQ5-SQ9 平台兼容性
- [ ] 破折号断裂句式0命中
- [ ] 每镜prompt字段完整（竖屏七法/光影/材质/尺度/运镜）
- [ ] 爆点/细节标注达标
- [ ] 暗线四线每集末尾完整

## 跨体裁SQ审查适配指南

> **背景**：短剧大师SQ体系最初基于修仙玄幻（封印/代价/灵脉系统）设计。  
> **《星河里唐宫夜宴》案例**（古穿今+美食+爱情）：60集，SQ总分61/120（C级），验证了体裁差异对评分的影响。

### 体裁差异处理表

| SQ关卡 | 修仙玄幻默认 | 古穿今/美食/爱情替代指标 | 影响 |
|:------:|-------------|------------------------|:----:|
| SQ1 | 同（E01前15秒信息差爽感） | **同** — 穿越题材同样适用 | 不变 |
| SQ5 | 暗线A-D（封印/代价/灵脉/反派） | 改为：**情感线+穿越真相线+反派升级线+事业线** | 🔴 高 |
| SQ6 | 代价曲线（%递减） | 改为：**穿越风险/代价**（留下会怎样？触发条件是什么？）或标记N/A | 🟡 中 |
| SQ8 | 版本统一（角色团） | **同** — 命名一致性（注意：苏瑶/苏媛命名不一致） | 不变 |
| SQ9 | 小云雀prompt格式 | **同** — 无论什么体裁，文学格式都不可直接提交 | 不变 |
| SQ11 | 反派叠压（封印突破级） | 改为：**反派梯度**（商业/情感/身体攻击的层次升级） | 🟡 中 |

### 审查前体裁分类步骤

审查60集前，先确认体裁特征以调整SQ侧重点：

```
□ 修仙玄幻 → 默认SQ体系（封印/代价/灵脉全部检查）
□ 古穿今/穿越 → SQ5用四线替代指标；SQ6标记N/A或解析为"穿越代价"
□ 现代言情/霸总 → 侧重SQ2尾钩、SQ11反派、SQ12情感闭环；SQ5/SQ6通常N/A
□ 都市商战 → 侧重SQ4逻辑（商业逻辑自洽）、SQ11反派叠压（降维打击层级）
□ 年代/家庭 → 侧重SQ3角色一致性、SQ12情感闭环
```

### 缺省项的处理规则

当某SQ关卡对当前体裁不适用时：

1. **显式声明N/A** — 不要跳过不写（否则报告少了一项，读者会疑惑）
2. **附替代评估** — 如"SQ6 代价曲线：N/A。替代评估：女主穿越风险代价不明确（触发条件=日食+玉佩，但留下会怎样无代价设计）→ 情感张力打折"
3. **在综合分中排除N/A项** — N/A项不计入分母。如120分体系中N/A 1项，则按110分计算

### 例子：古穿今美食爱情评分表头

```
| SQ5（暗线） | 替代指标 | 评分 | 评语 |
| 修仙默认 | 封印完整度 | — | N/A，本剧无此系统 |
| 古穿今替代 | 情感线/穿越真相线/反派升级/事业线 | 4/10 | 无结构化追踪，情感线摇摆过多 |
| SQ6（代价） | 穿越风险/留下代价 | N/A | 本剧无代价机制，但替代评估揭示缺憾 |
```

### 重要：不同体裁的修复优先级也不同

| 体裁 | P0修复项 | 原因 |
|------|---------|------|
| 修仙玄幻 | SQ5暗线连续性 + SQ6代价曲线 | 核心机制断裂 = 叙事崩塌 |
| 古穿今 | SQ1 E01爽点 + 穿越逻辑自洽 | 穿越锚点是观众信任基础 |
| 现代言情 | SQ2尾钩 + SQ12情感闭环 | 观众为情绪买单 |
| 商战 | SQ4叙事逻辑 + 商业合理性 | 商战观众对逻辑漏洞敏感 |



## 常见陷阱清单

| # | 陷阱 | 检测 | 修复 |
|---|------|------|------|
| 1 | 数值不可逆回退 | diff相邻集暗线值 | 逐集修正 |
| 2 | 破折号断裂句式 | regex扫描 | 改标准中文 |
| 3 | 版本断裂 | 比对角色团列表 | 二选一重写 |
| 4 | 代价斜率异常 | 检查单集跳变幅度 | 补情节支撑 |
| 5 | prompt字段渐进弱化 | 逐镜扫字段 | 全量补全 |
| 6 | 世界观过晚揭示 | 标记首现集号 | 提前分散 |
| 7 | 格式突变 | 比对相邻集尾钩 | 二选一统一 |
| 8 | 代价%与暗线A混淆 | 检查交叉引用 | 明确定义 |
| 9 | **E01首集无爽点** 🔴 | 前15秒无"观众独享信息差"时刻 | 注入爽点（§1.3.1a·方案A/B/C） |

## 第三方剧本评估模式（非自创剧本审查）

> **实战案例**：2026-06-15 对《星河里唐宫夜宴》（60集古穿今美食爱情 .docx 剧本）执行第三方评分。  
> **关键差异**：审查者不是作者，**无法提问创作意图**，只能从文本推断。

### 与自创审查的区别

| 维度 | 自创审查（内部QA） | 第三方评估（外部评分） |
|------|:-----------------:|:---------------------:|
| 能否修改原文 | ✅ 可以逐集修复 | ❌ 只评分不修改 |
| 暗线标注 | ✅ 文本中已有结构化标注 | ❌ 需从叙事文本中**反向提取** |
| 创作意图 | ✅ 作者可知 | ❌ 只能从文本推断 |
| 交付物 | 逐集修复后的版本 | **综合评分报告文档** |
| 报告格式 | 问题清单 | **SQ评分表+各维度分析+逻辑漏洞明细** |

### 前置步骤（专属SQ0）

读入外部剧本后，**先执行体裁分类**再评分：

```
1. 提取剧本文本（.docx → python-docx，.txt → read_file）
2. 通读前5集 + 每10集采样1集 → 确定体裁
3. 按体裁适配表（见§跨体裁SQ审查适配指南）调整SQ侧重点
4. 建立「替代指标映射表」记录哪些SQ换了评估维度
5. 然后开始逐SQ评分
```

### 第三方评估报告标准结构

```
# 《剧名》SQ评分报告

## 一、基本信息（体裁/集数/角色/世界观）
## 二、SQ评分总览（13行 × 分数/判定/评语）
## 三、各关卡详细分析（SQ0-SQ12，逐关200-500字）
## 四、亮点总结（纵然评分不高，仍值得肯定的点）
## 五、逻辑漏洞清单（按🔴P0→🟡P2→🟢P3排序，含位置和严重度）
## 六、升级路径（如果作者想改，需要多少工作量）
```

### 评分报告「交付确认」铁律

第三方评估的最终交付物是**文档**（不是聊天回复），写完文档后：

```
1. ✅ 写入桌面/项目目录下的专用文件夹
2. ✅ 同时输出 .docx 版（用户习惯双击打开）
3. ✅ 通过 open 命令在Finder中打开文件夹
4. ❌ 不要只说"文件在XXX路径"，用户可能找不到
```

### 第三方评估特有问题清单

| # | 问题 | 说明 |
|---|------|------|
| 1 | **命名不一致从文本推断** | 如"苏媛→苏瑶"，标注"可能的命名错误"而非确定性断言 |
| 2 | **无Phase 1元数据** | 无法判断正式版/旧版，报告中标"未知" |
| 3 | **暗线只能提取不能验证** | 标注"可提取"，保留审查者与作者意图可能不一致的余地 |
| 4 | **逻辑漏洞可能是体裁惯例** | 穿越剧的逻辑宽松度高于商战剧，扣分前确认体裁基准 |
| 5 | **无法判断AI改写还是人工** | docx文件可能由AI生成，推理逻辑时留有余地 |

## 并行策略

- 3路delegate_task并发（Hermes默认上限）
- 每路≤20集（子代理token预算内）
- 子代理context含完整审查标准和检测正则
- 用中文输出
