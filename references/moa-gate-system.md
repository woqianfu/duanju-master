## 二、四闸门协议

### G1 剧本质量闸 — 剧本大师联合MoA

| 属性 | 值 |
|:----|:----|
| 触发时机 | 剧本大师完成初稿后，**SQ0前** |
| MoA 任务 | 4模型交叉审剧本：GPT审框架、DeepSeek审执行、Claude聚合判分 |
| 输出 | 《MoA剧本预检报告》— 评分 + TOP3问题 + 修改建议 |
| 通过标准 | **≥7/10** 或用户确认"直接提交" |
| 失败处理 | 退回剧本大师修改，修改后重审（免MoA，直接改） |
| 费用 | ~$0.15/集 → 60集 = **$9.00** |

**MoA 剧本检查清单**：
```
□ 赛道定位是否清晰（古装/现代/仙侠？）
□ 人物功能位是否完整（主角/反派/助攻/误判者）
□ 第一屏压力是否足够
□ 前3集钩子密度合理
□ 长线结构（暗线A/B/C/D）有无断裂
□ 尾钩类型是否轮换
□ 20维质量矩阵是否有致命项
```

### G2 Prompt提交前闸 — 废片大师联合MoA ⭐ ROI最高

| 属性 | 值 |
|:----|:----|
| 触发时机 | **每次提交视频生成Prompt前**（废片大师+运镜大师+灯光大师输出后） |
| MoA 任务 | 4模型审Prompt：抓盲点、找矛盾、补漏洞、减废片率 |
| 输出 | 《MoA Prompt预检报告》— 缺陷清单 + 修复建议 + 信心分 |
| 通过标准 | 信心分 ≥ 0.8 |
| 失败处理 | 降废片大师修复 → 重审（免MoA）→ 提交 |
| 费用 | ~$0.003/镜（轻量级，仅Prompt文本）→ 60集×12镜 = **$2.16** |
| **ROI** | **废片率 15%→6%**, 每降1%省$5.76/剧 |

**MoA Prompt 检查清单**：
```
□ 运镜关键词是否单一无污染
□ 灯光六要素是否完整（Key/Fill/Backlight/色温/风格/方向）
□ 负向约束 ≤5条
□ 角色锁定（首帧锚定法是否生效）
□ 场景一致性（与前镜/前集服装/道具/环境一致）
□ 镜头分级（S/A/B/C 是否正确）
□ 时长/帧率/分辨率参数正确
□ 是否有导致废片的指令冲突
```

### G3 镜头分级闸 — 省钱大师联合MoA

| 属性 | 值 |
|:----|:----|
| 触发时机 | 剧本分镜完成后，**80/15/5 预算分配前** |
| MoA 任务 | 4模型从20维矩阵选出「最值得精修的3个镜头」 |
| 输出 | 《MoA分级建议》— 精修镜头TOP3 + 推荐档次 + 预估成本 |
| 通过标准 | 用户确认 |
| 失败处理 | 省钱大师改方案 → 再确认 |
| 费用 | ~$0.15/集 → 60集 = **$9.00** |
| **价值** | **精修预算利用率↑50%**（MoA选镜头 vs 凭感觉选） |

**MoA 分级判断矩阵**：
```
20维质量矩阵 × 镜头重要性 × 成本预算
  ↓
评分: 基础=🟢, 标准=🟡, 精修=🔴
  ↓
推荐: 最佳3个🔴镜头 + 建议分配 + 预估总成本
```

### G4 交付后质检闸 — 质检大师联合MoA

| 属性 | 值 |
|:----|:----|
| 触发时机 | SQ11 生成后质检 → 用户审阅前，**MoA终审** |
| MoA 任务 | 4模型做最后一轮一致性交叉验证 |
| 输出 | 《MoA终审报告》— 通过/有条件通过/不通过 + 剩余问题 |
| 通过标准 | ≥8/10 |
| 失败处理 | 回SQ重生成 → MoA再检 |
| 费用 | ~$0.15/集 → 60集 = **$9.00** |
| **价值** | **漏检率↓70%**, 防止交付后才发现问题 |

**MoA 终审检查清单**：
```
□ 跨集角色一致性（服装/发型/脸型）
□ 跨集场景一致性（布局/道具/光照方向）
□ 色彩基调统一
□ 运镜类型无突变
□ BGM与情绪匹配
□ 字幕/旁白对齐
□ 时长是否符合65s节奏版
□ 是否有未修复的SQ警告项
```

## 四、与22大师体系的协作方式

### 4.1 各大师与MoA的对接

| 大师 | MoA角色 | 协作方式 |
|:----|:--------|:--------|
| **案例大师** 🪖 | MoA分析对标案例 | 案例大师提供候选 → MoA做7维拆解 → 产出案例机制 |
| **剧本大师** | G1被审 | 剧本大师写初稿 → MoA审 → 报告给剧本大师修改 |
| **废片大师** | G2主审 | 废片大师出Prompt → MoA审 → 废片大师修复 |
| **省钱大师** | G3主审 | 省钱大师出预算方案 → MoA选精修镜头 → 省钱大师确认 |
| **质检大师** | G4主审 | 质检大师走SQ1-11 → MoA终审 → 合并报告 |
| **矩阵大师** | 质量参考 | 20维矩阵 → MoA在G2/G4调用质量维度 |
| **运镜大师** | G2参考 | 运镜关键词 → MoA检查污染 |
| **灯光大师** | G2参考 | 灯光六要素 → MoA验证完整性 |
| **进化大师** 🧬 | KPI监督 | 进化大师统计每个闸门的通过率/拦截率/节省成本 |

### 4.2 调用方式

| 场景 | 指令 |
|:----|:----|
| 手动单集MoA闸 | `/moa 检查这一集剧本质量，执行G1协议` |
| 批量全剧MoA闸 | `/moa 执行G2协议，检查所有720个Prompt` |
| 回顾MoA效果 | 「进化大师，查MoA闸门的本周拦截率和节省成本」 |
| 调闸门配置 | 「把G2的通过标准从0.8调到0.7」 |

### 4.3 自动化路径（触发链）

```

  → 通过后进SQ0-SQ2
```

## 七、局限性

1. **MoA 非实时** — 每次调用增加数秒延迟，不适合批量极速出片
2. **参考模型质量依赖** — 若GPT-5.5/DeepSeek V4降级，MoA质量随之下降
3. **G2 覆盖面** — 仅审Prompt逻辑，不审画面实际效果（废片率仍由生成模型决定）
4. **G4 不能替代人工终审** — MoA终审辅助，人工最终签字
5. **基准数据依赖** — 废片率基线（15%）和节省数据需定期校准
