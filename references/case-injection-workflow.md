# 案例注入工作流 — 案例大师执行手册

> 当发现外部新技术/工具/竞品/趋势与短剧大师相关时，案例大师按此流程注入案例知识。
> 来源：2026-06-28 OpenMontage 案例注入实操提炼。

---

## 六步注入流程

```
发现新案例（技术/工具/竞品/趋势）
  │
  ├─ Step 1：研究案例核心指标
  │   ├─ 成本/性能/架构等量化数据
  │   └─ 与短剧大师已有能力的对标点
  │
  ├─ Step 2：创建 reference 文档
  │   ├─ references/<case-name>.md
  │   ├─ 包含：指标/架构/可迁移机制/局限性
  │   └─ 格式：对标短剧大师大师体系，给出具体迁移建议
  │
  ├─ Step 3：注入 SKILL.md 参考列表
  │   └─ 在 SKILL.md 头部参考资料区域加一行
  │       > 📁 <分类标记>：`references/<case-name>.md`（一句话摘要）
  │
  ├─ Step 4：注入对应大师职责
  │   └─ 在相关大师的介绍中加入一句引用
  │       > 🔬 <分类标记>：`references/<case-name>.md` — <一句话触发条件>
  │
  ├─ Step 5：注入自进化矩阵
  │   ├─ references/masters-self-evolution.md
  │   ├─ 更新相关大师的搜索方向 + 穷举关键词 + 优化动作
  │   └─ ⚠️ 坑点：插入新行时，表格后面所有编号必须+1
  │
  └─ Step 6：通知进化大师
      └─ 本案例作为上新率记录，计入本次进化KPI
```

---

## 坑点备忘

| 坑 | 表现 | 修复 |
|----|------|------|
| 🚩 表格编号断链 | 注入新行后后续编号未更新 | 插入行后扫一遍全部编号，从新行起每行+1 |
| 🚩 未写局限性 | 用户发现片面引用 | 每篇参考文档必须包含「局限性」章节，诚实评估 |
| 🚩 参考链接重复 | 同一案例在不同大师引用多次但路径不同 | 统一用 `references/<case-name>.md`，所有大师引用同一条路径 |
| 🚩 SKILL.md 超限 | 文件超100K字符无法 patch | 改为只加一行引用链接，完整内容放 reference 文件 |
