# 大师合约表模板 (Master Alignment Table)

> **版本**：v6.4.0 新增
> **定位**：Phase 2.5 大师排期会的核心产物。每集一个合约表，承载22位大师对本集每场戏的场景级决策。
> **格式**：YAML风格，位于每集脚本文件首部，`---` 分割线之前。

---

## 三、冲突检测规则

> 在 Phase 2.5 排期会结束时，自动执行以下冲突检测。

| 冲突对 | 检测规则 | 触发条件 | 修复建议 |
|:------|:--------|:---------|:---------|
| 灯光 vs 运镜 | 光向VS运镜路径 | 运镜路径导致主体离开光源方向 | ①改运镜保持光向 ②改灯光布光方案适应运镜 |
| 构图 vs 运镜 | 七法VS运镜类型 | 运镜与构图效果冲突（如推镜+留白无效） | ①换运镜 ②换七法 |
| 肌肉 vs 对白 | 表情VS对白情绪 | 表情标签与对白情绪层级矛盾 | ①调整表情 ②调整对白措辞 |
| 配乐 vs 节奏 | BPMVS场景情绪 | 节奏标注抒情但配乐BPM>100 | ①换BGM ②换场景节奏标注 |
| 转场 vs 连贯 | 转场VS锚点连续性 | 硬切处道具/服装在锚点表有连续性需求 | ①改为叠化 ②拆分两镜使硬切可执行 |
| 暗线 vs 对白 | 暗线保密VS对白暴露 | 对白透露了暗线大师标注应保密的信息 | ①改对白 ②推迟本集暗线揭示 |
| 武术 vs 运镜 | 武打L2-L3VS运镜选择 | L3武打用大全景（看不到细节） | ①运镜换跟拍/特写 ②降低武打级别 |
| 灯光 vs 色调 | 灯光色温VS场景情绪 | 冷色温配温馨情绪的冲突 | ①换暖色调 ②调整情绪标注 |
| 省钱 vs 效果 | 省钱约束VS技术需求 | L3武打需要30fps但省钱要求24fps | ①L3镜头单独升30fps ②降低武打级别 |

## 四、排期会工作流程

```
Phase 2 产出标准化剧本
  │
  ▼
Phase 2.5 大师排期会
  │
  ├─ 第1步：剧本大师分场景展示剧本（本集所有场景+角色+对白+动作）
  │
  ├─ 第2步：各大师并行标注 — 每人填写合约表中自己负责的字段
  │      │
  │      ├─ 灯光大师：光向/色温/主辅轮（附录O）
  │      ├─ 运镜大师：主运镜/景别/速率（附录J.3.2）
  │      ├─ 构图大师：竖屏七法/站位/大气（竖屏七法）
  │      ├─ 肌肉大师：表情弧线（附录H）
  │      ├─ 配乐大师：BGM类型/BPM/音色（SQ7）
  │      ├─ 转场大师：前后转场类型（附录P）
  │      ├─ 连贯大师：道具锚点/服装/光向锁定（附录Q）
  │      ├─ 暗线大师：四线数值
  │      ├─ 武术大师：分级/动作拆分
  │      ├─ 对白大师：情绪层级/密度
  │      ├─ 造型大师：服装版本
  │      ├─ 省钱大师：技术参数
  │      └─ 废片大师：负面约束清单
  │
  ├─ 第3步：冲突检测 — 自动跑上表9种冲突
  │      │
  │      ├─ 无冲突 → ✅ 通过
  │      └─ 有冲突 → 提示涉及的大师协商 → 重填冲突字段
  │
  ├─ 第4步：合并 ➜ 产出大师合约表（写入脚本首部）
  │
  ├─ 第5步：合约确认 → 剧本大师确认无误 → 进入Phase 3
  │
  ▼
Phase 3（逐镜提交）
  → 每镜prompt从合约表提取对应字段
  → 合约表写入脚本文件首部，提交前可见
```

## 五、与逐镜prompt的映射关系

| 合约表字段 | 逐镜prompt位置 | 转换示例 |
|:----------|:--------------|:---------|
| 灯光.基调+色温 | 【场景/光线】前半 | `月光冷调#2D5A7B` |
| 灯光.主辅轮 | 【场景/光线】后半 | `key=moon45° fill=漫反 back=轮廓` |
| 运镜.主运镜+速率 | 【主运镜】 | `推：慢速·3s→6s` |
| 肌肉.表情弧线 | 【核心动作】后半 | `表情：惊讶(0-2s)→沉思(2-6s)` |
| 构图.竖屏七法 | 隐式（不写进prompt，由Agent确保） | — |
| 构图.大气镜头 | 隐式（由Agent确保画面有气氛） | — |
| 废片.负面约束 | 【负面约束】 | `no flickering, stable lighting` |
| 省钱.帧率 | submit_run参数 | `fps=24` |
| 省钱.Seed固定 | submit_run参数 | `seed=42` |
| 连贯.道具锚点 | 校验（写进20轮自检清单） | — |

> **原则**：合约表是「设计意图」，逐镜prompt是「执行结果」。合约表和prompt的映射可以是1:N（一个合约约束多个镜头），也可以是N:1（多个合约约束一个镜头）。
