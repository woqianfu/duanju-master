# 研究军团·案例大师团队工作流

案例大师(军团长) → 天网猎手(全网采集) + 数据分析师(趋势计算) 
→ 拆解师(七维拆解) + 机制提炼师(共性迁移) 
→ 竞品研究员(对标) + 趋势预言家(赛道预测)
→ {完整案例分析包} → 剧本大师 → 开始创作

详见 SKILL.md 「研究军团 · 案例大师团队」章节。

## 📥 案例注入工作流（六步法）

> 当发现外部新技术/工具/竞品/趋势与短剧大师相关时，按此流程注入。
> 详细执行手册 → `references/case-injection-workflow.md`

| 步骤 | 动作 | 产出 | 目标文件 |
|:----:|------|------|---------|
| 1 | 研究案例核心指标 | 成本/架构/对标点 | — |
| 2 | 创建 reference 文档 | 完整案例分析 | `references/<case>.md` |
| 3 | 注入 SKILL.md 参考列表 | 一行引用 | SKILL.md 头部 |
| 4 | 注入对应大师职责 | 一行触发条件 | 相关大师章节 |
| 5 | 注入自进化矩阵 | 搜索方向+关键词 | `references/masters-self-evolution.md` |
| 6 | 通知进化大师 | 计入KPI | 进化报告 |

**⚠️ 坑点**：第5步插入新行后，表格后面所有编号必须+1；每篇 reference 必须包含「局限性」章节。
