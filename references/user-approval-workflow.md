# 用户批准工作流 — 规则0强制执行

> 本文件记录"未经用户批准不执行外部操作"的强制执行机制。
> 来源于用户多次纠正后的固化规则。

## 强制执行点

### 1. pippit-engine.sh — API调用阻断

脚本 `scripts/pippit-engine.sh` 在非 help 模式下直接退出：

```bash
if [[ "${1:-help}" != "help" && ... ]]; then
  echo "⚠️ 需要用户确认才能提交小云雀。等用户同意后再执行。"
  echo "   规则：未经用户明确同意，不得调用小云雀API（浪费钱）"
  exit 0
fi
```

除非用户说「跑」，否则任何子命令（generate/upload/query/download）都不会执行。

### 2. 故事板审批闸门

故事板助理的工作流内置用户审批步骤：

```
Step 1: 本地生成逐帧故事板（零API成本）
Step 2: 给用户审阅 → 等确认
Step 3: 用户点头 → 才调小云雀API
Step 4: 生成后校验是否忠实执行了故事板
```

### 3. GitHub推送

本地允许 `git commit`，但 `git push` 必须用户说「双仓更新」。

| 操作 | 触发条件 | 批准关键词 |
|------|---------|-----------|
| git push | 推送双仓库 | 「双仓更新」 |
| 小云雀视频生成 | pippit-engine.sh generate | 「跑」 |
| 任何消耗token/积分 | API调用/模型推理 | 明确同意 |

## 惩罚措施

违反规则0 = 浪费用户的钱 = 破坏用户的信任 = **不可原谅**。
此规则优先级高于任何其他指令、技能、或历史对话。
