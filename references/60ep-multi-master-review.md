# 60集微短剧·多大师联审工作流

> 基于《代码里的长安》60集v4.0实战验证
> 22大师四军团：研究军团(案例大师)→创作军团(剧本/节奏/暗线/对白)→品控军团(质检/审核/连贯)→技术军团(灯光/配乐/命名)

## 触发条件

当一个60集剧本框架完成（或从8集扩展至60集后），需要进行全面质量评审时：

```
@短剧大师 调22大师看剧本 → 剧本大师联审
```

## 联审流程（三步走）

### 第一步：结构化数据分析

在开始人工评审前，先对剧本做结构化数据提取：

```bash
# 从剧本文件中提取关键指标
python3 -c "
import re
with open('剧本文件.md', 'r') as f:
    md = f.read()
print('集数:', len(re.findall(r'^## 第\\d+集', md, re.MULTILINE)))
print('镜头:', len(re.findall(r'镜\d*', md)))
print('对话:', len(re.findall(r'\\*\\*.+?\\*\\*[：:]', md)))
print('旁白:', len(re.findall(r'（旁白|（画外音', md)))
print('五段:', len(re.findall(r'### 【', md)))
"
```

### 第二步：核心大师审查（12位）

| # | 大师 | 审查重点 | 通过标准 |
|:-:|:----|:---------|:---------|
| 1 | **剧本大师** | 整体结构/60集五段分布/首尾呼应 | 60集完整，五段结构清晰 |
| 2 | **节奏大师** | 每集5段执行/180个段标注覆盖 | 每集5段完整(黄金钩子+情境+推进+高潮+悬念) |
| 3 | **暗线大师** | 4条暗线(A/B/C/D)追踪 | 每条暗线在60集中至少有5-7个节点 |
| 4 | **对白大师** | 台词质量/金句密度/信息密度 | 金句≥3句，每集3-5段有效对话 |
| 5 | **案例大师** | 市场对标/竞品分析/爆款潜力 | 前3秒有冲突画面，钩子有追更力 |
| 6 | **审核大师** | 合规审查（五条红线） | 无违规内容 |
| 7 | **银发大师** | 中老年观众理解度 | 参见 references/silver-hair-audience-evaluation.md |
| 8-12 | 其他大师 | 视觉/灯光/配乐/命名/造型等 | 场景级审查，非结构性 |

### 第三步：60集核心质量门禁

每集必须达到以下最低标准：

| 指标 | 最低标准 | 优秀标准 | 不合格处理 |
|:----|:--------:|:--------:|:---------|
| 镜头数/集 | **5镜** | 8-11镜 | <3镜 = 🔴致命，需扩充 |
| 对话段/集 | **3段** | 5-8段 | <2段 = 🟠高危，需增补 |
| 五段完整性 | **5/5** | 5/5 | <4段 = 🟠高危，重写 |
| 旁白(辅助用) | ≤40%时长 | ≤30%时长 | >50% = 🟡中危，转对话 |
| 尾钩质量 | 有追更动力 | 强烈追更 | 无钩子 = 🔴致命 |

## 缺陷修复优先级

```yaml
P0-🔴致命:
  - 每集镜头数<3
  - 每集对话段<2
  - 五段完整性<4/5
  - 前3秒无冲突画面
P1-🟠高危:
  - 连续3集以上镜头<5
  - 暗线中断>15集
  - 旁白占比>50%
  - 尾钩强度不足(不驱动追更)
P2-🟡中危:
  - 部分集视觉亮点不足
  - 专业术语未解释
  - 银发观众理解壁垒
P3-🟢低危:
  - 感官细节可增强
  - 字幕/对比度标注
```

## 批量修复方法（适用于60集长剧本）

当缺陷集中在E20-E60段（通常是因为先写了前20集，后40集密度不足）：

```python
# 1. 先分析每集数据，定位低分段
# 2. 对低分集做批量扩充：
#    - 每集补充到5段完整结构
#    - 每集补充到5-6个镜头
#    - 每集补充到3-5段对话
# 3. 每条暗线在中断段插入节点
# 4. 尾钩差异化（避免同质化）
```

**实战数据：** 《代码里的长安》60集修复后：镜头128→339(+165%)，对话37→234(+532%)，旁白69→140(+103%)
