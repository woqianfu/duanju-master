# 案例大师·研究军团设计档案

## 设计背景

2026-06-17 用户提出：案例大师不能只有一个人，需要团队。

## 编制

| # | 角色 | 职责 |
|---|------|------|
| 1 | **案例大师** 🪖 军团长 | 全局指挥+报告总成 |
| 2 | **天网猎手** 🕸️ | 7×24h 全网并行采集（抖音/快手/微信/微博/小红书） |
| 3 | **数据分析师** 📊 | 数据清洗+趋势计算（完播率/弹幕密度/CPM） |
| 4 | **拆解师** 🔍 | 每案例七维拆解（赛道→机制→节奏→视觉→受众→数据→可迁移） |
| 5 | **机制提炼师** ⚙️ | 跨案例找共性机制，写迁移说明书 |
| 6 | **竞品研究员** 🎯 | 对标分析+差异化切入点 |
| 7 | **趋势预言家** 🔮 | 赛道生命周期判断（上升/成熟/衰退） |

## 并行三路

- Phase 1: 天网猎手 + 数据分析师 → 《数据采集报告》
- Phase 2: 拆解师 + 机制提炼师 → 《案例拆解包》
- Phase 3: 竞品研究员 + 趋势预言家 → 《竞争情报包》
- 案例大师合并三包 → {完整案例分析包} → 剧本大师

## 与创作军团衔接

案例大师团队输出 → 剧本大师参照案例机制创作 → 二十大师会审时案例对照
