# 拍摄级剧本格式规范（60秒/集）

> 2026-06-27 从《代码里的长安》60集重写实战中提炼
> 解决的核心问题：旧版剧本每集只有1-2句描述拍不满60秒；缺少旁白导致过渡生硬

## 一、核心原则

| 原则 | 说明 |
|------|------|
| **60秒闭环** | 每集必须有足够内容拍满55-65秒，不允许1-2句话糊弄 |
| **旁白不可缺** | 没有旁白的剧本就像没有润滑剂的齿轮——场景之间的过渡会非常生硬 |
| **5段结构** | 每集严格执行：黄金钩子→情境定位→推进→高潮→悬念 |
| **镜头数>=8** | 每集至少8个镜头，平均10-11个，确保信息密度 |
| **时间码标注** | 每个镜头标注起止时间（如 0-3s, 22-28s） |

## 二、每集5段式结构

```
00:00 ---- 05 ---- 15 ---- 35 ---- 50 ---- 60
  ↑         ↑       ↑       ↑       ↑
  黄金钩子 情境定位 推进冲突 高潮点  悬念收尾
```

### 1. 黄金钩子（0-5s）
- 第一帧就抓住观众，不能是风景空镜或上集回顾
- 直接扔冲突/危机/高能画面

### 2. 情境定位（5-15s）
- 10秒内让观众知道这是哪、谁在干嘛、当前状态
- 用动作+对话+画面自然带出，不要用旁白硬交代

### 3. 推进/冲突（15-35s）
- 20秒内完成一个小循环：遇到问题-尝试解决-遇到新问题
- 每5秒至少有一个小起伏

### 4. 高潮点（35-50s）
- 本集情绪峰值——最燃/最暖/最炸的瞬间
- 要有冲击力的画面+一句话金句

### 5. 悬念收尾（50-60s）
- 五种钩子型：危机迫近/信息炸弹/身份揭露/倒计时/悬念对话
- 不能虚、不能剧透、必须有下集必须看的理由

## 三、单镜头格式规范

镜号 + 时间码
场景描写：环境+人物状态+动作
（旁白）背景/过渡/情绪点题

角色名
对白

## 四、常见陷阱

### 陷阱1：剧情大纲冒充剧本
现象：每集只有一句话概括（"十七说出秘密""何砚秋来了"）
后果：拍不了、导演拿到剧本不知道拍什么
修复：展开成8-11个镜头，每个镜头有具体画面描写

### 陷阱2：没有旁白
现象：场景直接切换，观众不知道时间/地点/人物状态发生了什么变化
后果：过渡生硬，情感断层
修复：每集至少3-5段旁白，交代时间变化、人物内心、背景知识

### 陷阱3：信息密度不足
现象：一集只有一两句对白就结束了
后果：拍不满30秒，观众觉得"就这？"
修复：每集至少安排5组以上的对白/动作/反应交互

### 陷阱4：时间码虚标
现象：写了时间码但实际内容只够10秒
后果：导演无法按时间码排期
修复：写完拿给第三方读一遍——"这段读下来要几秒？"不够就加

## 五、自查清单

每集写完回答这5个问题：
1. 这一集能不能拍满60秒？
2. 没有旁白只看画面能不能看懂剧情？
3. 前3秒有没有让人想继续看的东西？
4. 最后10秒有没有让人想看下一集的钩子？
5. 这一集单独拿出来好看吗？

任何一题答"不能"——这集需要重写。
